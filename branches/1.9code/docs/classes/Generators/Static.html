Class Generators::Static
In: src/generators.rb
Parent: Generator

In: Value, Limit, Transform1, … TransformN

Out: Value, passed through each Transform in order

Although named Static, this generator can be passed Proc objects that refer to variables in the appropriate namespace, which can allow the output to change. If you update a variable that affects a Proc you will need to make one call to next to clear the last queue entry before the output stream changes. Proc objects that make internal calls to Kernel::rand will also produce a variable output stream.

The Generator will run out after Limit calls, or never if Limit is specified as -1.

WARNING: Don‘t pass one of these to the Cartesian generator with a limit of -1 or you‘ll get an infinite loop.

Example: 

 g=Generators::Static.new("angry gibbon", 5, proc {|s| OpenSSL::Digest::MD5.new( s + rand(256).chr ) } )
 g.to_a
 => [e2fc714c4727ee9395f324cd2e7f331f, 68774090e020b81d2bd584298a8cd612,
     971f3cbaa55fa9f7290effc88b42b39b, dcc4a4d1992c0cd595454eb34b74e761,
     a7aef58ed131b9ebf095d608d57529d9]
